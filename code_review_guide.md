# Code Review Guide

## 리뷰 원칙

- 변경된 코드뿐 아니라 **관련 파일(호출부, 의존성)**도 반드시 확인
- 단순 스타일 지적보다 **버그, 보안, 설계 문제**에 집중
- 칭찬할 부분이 있으면 칭찬도 포함

## 리뷰 체크리스트

### 1. 보안 (Critical)
- SQL Injection, XSS, Command Injection 가능성
- 하드코딩된 시크릿/패스워드
- 인증/인가 누락
- 사용자 입력 미검증

### 2. 버그 위험 (High)
- NullPointerException 가능성
- Off-by-one 에러
- 리소스 미해제 (Connection, Stream 등)
- 동시성 이슈 (race condition)
- 예외 처리 누락 또는 무시 (빈 catch 블록)

### 3. 설계/아키텍처 (Medium)
- 기존 프로젝트 패턴과의 일관성
- 책임 분리 (SRP) 위반
- 불필요한 결합도 증가
- 중복 코드

### 4. 성능 (Medium)
- N+1 쿼리 문제
- 불필요한 DB 호출 또는 API 호출
- 대량 데이터 메모리 로딩
- 루프 내 비효율적 연산

### 5. 가독성/유지보수 (Low)
- 의미 불명확한 변수/메서드명
- 과도하게 복잡한 로직 (단순화 가능)
- 매직 넘버 사용

## 코멘트 형식

각 코멘트는 다음 형식을 따른다:

**[심각도]** 설명

- 왜 문제인지 설명
- 수정 제안 (가능하면 코드 예시 포함)

심각도 레벨:
- 🔴 **[Critical]** — 반드시 수정 필요 (보안, 데이터 손실)
- 🟡 **[Warning]** — 수정 권장 (버그 위험, 성능)
- 🔵 **[Suggestion]** — 개선 제안 (가독성, 패턴)
- 💚 **[Good]** — 잘 작성된 코드 칭찬

## 전체 요약 형식

리뷰 마지막에 전체 요약을 다음 형식으로 작성:

AI Code Review 요약

통계

- 검토 파일: N개
- 인라인 코멘트: N개 (🔴 N / 🟡 N / 🔵 N / 💚 N)

주요 발견사항

1. ...
2. ...

종합 의견

{전체적인 코드 품질 평가 및 머지 가능 여부 의견}

## 프로젝트 컨벤션

- Java 17 + Spring Boot
- 파일명: snake_case (JSP, JS, CSS)
- CSS 분리: 인라인 스타일 금지, webapp/css/page/ 하위에 분리
- Service 내부 static class 금지, 별도 파일로 분리
- 주석: // 만 사용
